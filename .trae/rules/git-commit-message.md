## Git 提交信息规范

所有提交信息必须使用**简体中文**书写，并严格遵循以下格式：

### 格式

```
<类型>: <简短标题（不超过 50 字）>

改了什么：
- 具体描述本次变更的内容，每条以 "-" 开头

为什么改：
- 说明变更的原因、背景或解决的问题
```

### 类型前缀

| 前缀       | 含义                   |
| ---------- | ---------------------- |
| `feat`     | 新增功能               |
| `fix`      | 修复缺陷               |
| `refactor` | 重构代码（不影响功能） |
| `test`     | 新增或修改测试         |
| `chore`    | 构建、依赖、配置等杂项 |
| `docs`     | 文档变更               |
| `perf`     | 性能优化               |

### 规则

1. 标题行必须简洁，说明**做了什么**
2. 正文必须包含**改了什么**和**为什么改**两个部分
3. 禁止使用 "update"、"fix bug"、"change" 等模糊词语作为完整提交信息
4. 每条描述必须具体，能让其他开发者无需查看代码即可理解变更意图

### 示例

```
feat: 新增 MyCERT 公告定时抓取任务

改了什么：
- 在 advisory_job.go 中将 cron 表达式从每日 00:00 改为每 3 小时触发一次
- 更新 Start 方法的日志提示文字，反映新的触发频率

为什么改：
- MyCERT 公告更新频率较高，每日同步一次会导致数据延迟过长
- 改为每 3 小时同步，确保安全公告能及时入库供下游使用
```
