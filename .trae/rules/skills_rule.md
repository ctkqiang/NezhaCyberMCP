## 技能引用规范

本项目所有开发工作必须参照并运用 `.trae/skills/` 目录中定义的技能档案。
该目录是本项目的技术能力基准，AI 在生成代码、提供建议、编写文档时，
必须以这些技能档案为依据，确保输出内容与项目技术栈保持一致。

---

### 当前已定义技能

| 文件               | 技能     | 熟练度 | 适用范围                                        |
| ------------------ | -------- | ------ | ----------------------------------------------- |
| `golang.md`        | Go Lang  | 90%    | 后端服务、API、定时任务、数据采集、网络安全工具 |
| `cybersecurity.md` | 网络安全 | 85%    | 漏洞研究、安全公告采集、渗透测试工具、数字取证  |
| `database.md`      | 数据库   | 80%    | PostgreSQL、MySQL、SQLite、GORM、事务管理       |
| `devops.md`        | DevOps   | 75%    | Docker、CI/CD、Git、环境配置、优雅关闭          |

---

### 强制要求

1. **代码生成**：生成任何 Go 代码时，必须遵循 `golang.md` 中定义的核心能力与最佳实践，
   包括并发模型、GORM 用法、HTTP 客户端封装、日志规范与测试方式。

2. **数据库操作**：所有数据库相关代码必须参照 `database.md`，
   使用 `WithContext`、`OnConflict`、`CreateInBatches` 等规范模式。

3. **安全功能**：涉及安全公告采集、漏洞数据处理、API 认证的代码，
   必须参照 `cybersecurity.md` 中的安全实践，不得硬编码凭据，不得忽略错误。

4. **部署与运维**：容器化配置、环境变量管理、进程信号处理，
   必须参照 `devops.md` 中的规范，确保服务可优雅关闭与多环境部署。

5. **技术文档**：编写任何技术说明、注释或文档时，
   必须引用对应技能档案中的术语与描述，保持表述一致性。

6. **新增技能**：当项目引入新技术栈时，必须在 `.trae/skills/` 目录下
   新建对应的技能档案，并在本文件的技能列表中补充注册。
